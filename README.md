# USB三麦克风录音上传系统

## 系统概述

本系统是为嘉立创泰山派 RTK3566 开发的USB三麦克风录音上传解决方案。系统包含录音上传客户端和接收服务端两部分，实现录音、上传、自动删除本地文件的完整流程。

## 文件结构

```
├── record_uploader.py    # 录音上传客户端
├── receiver.py           # 文件接收服务端
├── uploader.py           # 通用文件上传工具
└── radio.py              # 原始录音脚本（参考）
```

## 功能特点

### 录音上传客户端 (`record_uploader.py`)
- **多麦克风支持**: 同时支持三个USB麦克风(M1/M2/M3)录音
- **自动上传**: 录音完成后自动上传至指定服务器
- **自动清理**: 上传成功后自动删除本地文件，节省存储空间
- **设备持久化**: 设备配置自动保存，下次启动快速恢复
- **故障恢复**: 录音失败自动重试，连续失败自动切换设备
- **状态统计**: 实时显示录音、上传、失败统计

### 文件接收服务端 (`receiver.py`)
- **层级目录结构**: 按 IP地址 → 麦克风ID(M1/M2/M3) 组织文件
- **并发支持**: 支持多个客户端同时上传
- **进度显示**: 实时显示文件接收进度

## 安装与运行

### 前置依赖

**服务端**: Python 3.x (无需额外依赖)

**客户端 (泰山派)**: 
- Python 3.x
- ALSA 音频工具 (`arecord`)

```bash
# 安装ALSA工具
sudo apt-get update
sudo apt-get install -y alsa-utils
```

### 启动服务端

```bash
# 在接收文件的机器上启动服务
python3 receiver.py <端口>

# 示例
python3 receiver.py 8888
```

### 启动录音上传客户端

```bash
# 在泰山派设备上运行
python3 record_uploader.py -s <服务器IP> -p <端口>

# 示例
python3 record_uploader.py -s 192.168.1.100 -p 8888

# 自定义参数
python3 record_uploader.py -s 192.168.1.100 -p 8888 -d 30 -i 5
```

## 参数说明

### record_uploader.py 参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `-s`, `--server` | string | 必填 | 服务器IP地址 |
| `-p`, `--port` | int | 8888 | 服务器端口 |
| `-d`, `--duration` | int | 60 | 每次录音时长（秒） |
| `-i`, `--interval` | int | 1 | 录音间隔（秒） |
| `-o`, `--output` | string | usb_recordings | 临时文件目录 |

### receiver.py 参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `端口` | int | 必填 | 监听端口 |

```
│   │   ├── 20240101_120000_M2.wav

│   └── ...
```

## 使用场景

1. **多设备音频采集**: 多个泰山派设备同时录音并上传至中心服务器
2. **远程监控**: 无人值守环境下的音频采集和传输
3. **分布式录音**: 会议、演出等场景的多麦克风同步录音

## 技术细节

### 音频参数
- 采样率: 16kHz
- 位深: 16bit
- 声道: Mono (单声道)
- 格式: WAV

### 网络协议
- 传输协议: TCP/IP
- 数据分块: 4096字节
- 握手机制: READY → 数据 → OK

### 错误处理
- 录音失败自动重试（最多3次）
- 连续失败5次自动尝试切换设备
- 上传失败保留本地文件待重试

## 注意事项

1. 确保客户端和服务端在同一局域网内
2. 服务器端口需开放防火墙限制
3. 建议使用带电源的USB集线器连接多个麦克风
4. 首次运行会自动检测USB设备并生成配置文件
