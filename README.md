# Audio-Analysis｜智能声音收集与分析系统

> 通过部署声音传感设备，对火电除灰装置的爆炸/冲击效果进行音频采集与后续分析。

本仓库提供一套面向 **嘉立创泰山派 RTK3566** 的 USB 三麦克风录音上传方案。系统由录音上传客户端、文件接收服务端和通用上传工具组成，可完成多路 USB 麦克风录音、自动上传、服务端分目录保存以及上传成功后自动清理本地文件等流程。

## 功能特性

- **三路 USB 麦克风采集**：支持 M1/M2/M3 多麦克风同时录音。
- **自动设备检测与分配**：通过 ALSA 检测可用 USB 音频设备，并保存设备配置，便于下次启动复用。
- **录音后自动上传**：录制完成后通过 TCP 上传到指定服务端。
- **本地空间自动清理**：上传成功后删除本地临时音频文件；上传失败则保留文件，便于后续排查或重传。
- **故障恢复机制**：录音失败自动重试，连续失败时尝试切换设备。
- **服务端分类存储**：服务端按照 `客户端 IP / 麦克风编号` 自动归档文件。
- **通用文件上传工具**：`uploader.py` 可用于上传单个文件或文件夹。

## 运行环境

### 服务端

- Python 3.x
- 无第三方 Python 依赖

### 客户端（泰山派）

- Python 3.x
- ALSA 录音工具 `arecord`
- 1～3 个 USB 麦克风，建议使用带独立供电的 USB Hub

安装 ALSA 工具：

```bash
sudo apt-get update
sudo apt-get install -y alsa-utils
```

检查麦克风是否被系统识别：

```bash
arecord -l
arecord -L
```

## 快速开始

### 1. 启动接收服务端

在接收音频文件的服务器或电脑上运行：

```bash
python3 receiver.py 8888
```

服务端默认监听 `0.0.0.0:<端口>`，接收到的文件会保存到 `uploads/` 目录。

### 2. 启动录音上传客户端

在泰山派设备上运行：

```bash
python3 record_uploader.py -s <服务器IP> -p 8888
```

示例：

```bash
python3 record_uploader.py -s 192.168.1.100 -p 8888
```

自定义录音时长、录音间隔和临时目录：

```bash
python3 record_uploader.py \
  -s 192.168.1.100 \
  -p 8888 \
  -d 30 \
  -i 5 \
  -o usb_recordings
```

## 参数说明

### `record_uploader.py`

| 参数 | 类型 | 默认值 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| `-s`, `--server` | string | - | 是 | 接收服务端 IP 地址 |
| `-p`, `--port` | int | `8888` | 否 | 接收服务端端口 |
| `-d`, `--duration` | int | `60` | 否 | 单次录音时长，单位秒 |
| `-i`, `--interval` | int | `1` | 否 | 每轮录音之间的间隔，单位秒 |
| `-o`, `--output` | string | `usb_recordings` | 否 | 客户端本地临时录音目录 |

### `receiver.py`

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `端口` | int | 是 | 服务端监听端口，例如 `8888` |

### `uploader.py`

```bash
python3 uploader.py <服务器IP> <端口> <文件或文件夹路径>
```

示例：

```bash
# 上传单个文件
python3 uploader.py 192.168.1.100 8888 ./test.wav

# 上传整个文件夹
python3 uploader.py 192.168.1.100 8888 ./usb_recordings
```

> 注意：`record_uploader.py` 与 `receiver.py` 使用的上传头格式为 `mic_id|file_name|file_size`；`uploader.py` 是通用上传工具，上传头格式为 `file_name|file_size`。如需让 `uploader.py` 与当前 `receiver.py` 完全兼容，建议统一两端协议格式。

## 音频与传输参数

### 音频格式

- 采样率：16 kHz
- 位深：16 bit
- 声道：Mono 单声道
- 格式：WAV
- ALSA 录音命令核心参数：`S16_LE / 16000 Hz / 1 channel`

### 网络传输

- 协议：TCP/IP
- 分块大小：4096 bytes
- 交互流程：

```text
客户端发送文件头 -> 服务端返回 READY -> 客户端发送文件数据 -> 服务端返回 OK
```

## 常见问题

### 1. 检测不到 USB 麦克风

先检查系统是否识别设备：

```bash
arecord -l
arecord -L
```

如果没有设备：

- 检查 USB 麦克风是否供电正常；
- 尝试更换 USB Hub 或接口；
- 建议使用带独立供电的 USB Hub；
- 重启 ALSA 或重启设备后再试。

### 2. 客户端提示“无法连接到服务器”

请确认：

- 服务端 `receiver.py` 已启动；
- 客户端填写的服务器 IP 和端口正确；
- 客户端与服务端在同一网络内；
- 防火墙已放行对应端口，例如 `8888`。

### 3. 上传失败后文件去哪了？

上传失败时，客户端会保留本地 WAV 文件，不会自动删除。可在 `usb_recordings/M1`、`usb_recordings/M2`、`usb_recordings/M3` 下查找。

### 4. 录音文件大小异常

程序会根据录音时长估算合理文件大小。如果文件过小或过大，会判定为异常并删除该异常文件。常见原因包括麦克风掉线、设备被占用或 ALSA 录音中断。

## 适用场景

- 火电除灰装置爆炸/冲击效果的现场声音采集；
- 多设备、分布式音频采集；
